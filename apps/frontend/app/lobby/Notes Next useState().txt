NextJs weirdness
If we want to create a component for a page, that has a counter,
and a button to increase the counter, it looks like this:

	'use client';
	import { useState } from 'react';
	export default function Counter() {
		const [count, setCount] = useState(0);
		return (
			<div>
			<p>Count: {count}</p>
			<button onClick={() => {setCount(count + 1)}}>Increase</button>
			</div>
		);
	}

Now, going through this step-by-step:

'use client';
Is like a pre-processor instruction, which we have to do,
because our component has interactibility.
There are server and client components.
When a page is requested, Next goes to the page and its components and compiles them.
Server compoents can be compiled very well,
and if they fetch data from the backend, then they will do so during compilation.
This is great, because once the "binary" is compiled,
the clients browser doesnt have to go and fetch that data, as it was already included.
This is cool and great, but there is a major drawback.
Server components dont have interactibility,
meaning that you cannot have buttons or anything the user interacts with,
apart from loading the page once.
For things like buttons, changing elements,
and anything that isnt static, we need Client Components.
A component is market as such, by using the 'use client'; line at the top of the file.

import { useState } from 'react';

const [count, setCount] = useState(0);
We create a variable that holds a variable and a function.
We then call the useState() hook.
This goes into the useState()s internal memory,
and checks the data we have at the internal index.
If there is nothing there yet, set the data based on the useState()s parameters.
It then returns that data, which gets stored in our variable,
and the internal index is increased to the next slot.
We now have a variable and a function for that variable.
The variable is a constant deep copy of the data inside the memory slot,
that was accessed from calling useState().
The function is fixed to be a setter. Any parameter given to it,
will update the value at the slots internal memory.
So the value cannot be changed normally, as it is merely a copy.
To actually change the variable we use the setter function.

return ()
We then return some JSX, which is just javascript+html,
that will be used to render this component.

<p>Count: {count}</p>
We set a paragraph html element, with its value based on the variable,
from the data we retreived above.

<button onClick={() => {setCount(count + 1)}}>Increase</button>
Create a button element, that executes a javascript function on clicking.
The function calls the setter we retreived from the above useState() call,
with whatever new value we want to set the internal memory to.
